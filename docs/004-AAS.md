## 核心原则

### Answer First

Content 不是终点，Answer 才是终点。

所有回答必须遵循：**先给结论，再给证据，最后给来源。**

---

## 回答格式规范

### 通用结构

```
Answer = Conclusion + Evidence + Sources + Related Questions
```

| 部分 | 说明 | 必须 |
|------|------|------|
| **Conclusion** | 直接回答问题，一句话总结 | ✅ 必须 |
| **Evidence** | 支撑结论的事实列表 | ✅ 必须 |
| **Sources** | 原始来源链接，可追溯 | ✅ 必须 |
| **Related Questions** | 相关问题推荐 | ❌ 可选 |

## 回答类型规范

### Type 1: Lookup（查询）

**适用问题**："哪些 X 支持 Y？"、"X 有哪些 Y？"

**格式**：

```
Conclusion:
  {N} 个 {实体类型} 支持 {能力}

Evidence:
  - {实体1}: {支持的具体能力版本/说明}
  - {实体2}: {支持的具体能力版本/说明}
  - ...

Sources:
  - [{来源名}](url)
  - [{来源名}](url)

Visualization:
  Table（实体 × 能力）
```

**示例**：

```
Q: 哪些模型支持长期记忆？

Conclusion:
  4 个模型支持长期记忆能力

Evidence:
  - GPT-4: 支持跨会话长期记忆，最多保存 100 个对话
  - Claude 3.5 Sonnet: 支持 Project Memory 和 User Memory
  - Gemini 1.5 Pro: 支持长期记忆，需开启记忆模式
  - Qwen 2: 支持长期记忆，默认开启

Sources:
  - [OpenAI Blog](https://openai.com/news/gpt-4-memory)
  - [Anthropic News](https://www.anthropic.com/news/claude-memory)

Visualization:
  Table: Model | Memory Type | Notes
```

  Comparison Table:
    | Feature | X | Y |
    |---------|---|---|
    | {能力1} | {X的情况} | {Y的情况} |

Pros (X):
  - {优势1}
  - {优势2}

Cons (X):
  - {劣势1}

Pros (Y):
  - {优势1}
  - {优势2}

Cons (Y):
  - {劣势1}

Sources:
  - [{来源名}](url)
  - [{来源名}](url)

Visualization:
  Comparison Table / Radar Chart
```

**示例**：

```
Q: GPT-4 和 Claude 3.5 在能力上有什么差异？

Conclusion:
  GPT-4 在代码能力和多模态理解上更强，Claude 在长上下文和记忆能力上更优

Evidence:
  Comparison Table:
    | Feature | GPT-4 | Claude 3.5 Sonnet |
    |---------|-------|-------------------|
    | Context Length | 128k | 200k |
    | Code Generation | ⭐⭐⭐⭐ | ⭐⭐⭐ |
    | Vision | ⭐⭐⭐⭐ | ⭐⭐⭐ |
    | Memory | ⭐⭐ | ⭐⭐⭐⭐ |
    | Tool Use | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

Pros (GPT-4):
  - 更强的代码能力
  - 更准确的图像理解

Cons (GPT-4):
  - 上下文长度较短
  - 记忆能力较弱

Pros (Claude 3.5):
  - 更长的上下文
  - 更强的记忆能力

Cons (Claude 3.5):
  - 代码能力稍弱

Sources:
  - [OpenAI Blog](https://openai.com/research/gpt-4)
  - [Anthropic News](https://www.anthropic.com/news/claude-3-5-sonnet)

Visualization:
  Comparison Table / Radar Chart
```

### Type 4: Summary（总结）

**适用问题**："最近有什么重要变化？"、"趋势是什么？"

**格式**：

```
Conclusion:
  {核心总结，2-3句话}

Evidence:
  Key Points:
    1. {要点1}: {详细说明}
    2. {要点2}: {详细说明}
    3. {要点3}: {详细说明}

Trend Analysis:
  {趋势分析}

Sources:
  - [{来源名}](url)
  - [{来源名}](url)

Visualization:
  List / Chart
```

**示例**：

```
Q: 最近 90 天最热门的 AI 能力是什么？

Conclusion:
  最近 90 天最热门的能力是 Agent 自主执行和 MCP 支持，其次是长期记忆

Evidence:
  Key Points:
    1. Agent 自主执行: 新增支持的产品从 5 个增加到 12 个，增长 140%
    2. MCP 支持: 新增支持的产品从 3 个增加到 8 个，增长 167%
    3. 长期记忆: 新增支持的模型从 2 个增加到 4 个，增长 100%

Trend Analysis:
  Agent 能力正在成为竞争焦点，MCP 协议正在快速普及

Sources:
  - [Anthropic News](https://www.anthropic.com/news/mcp)
  - [Cursor Blog](https://cursor.sh/blog/agent-update)

Visualization:
  Bar Chart（能力增长对比）
```

## 回答质量标准

### 准确性

- 所有结论必须有证据支撑
- 证据必须可追溯到原始来源
- 不允许编造信息

### 完整性

- 回答必须覆盖问题的所有方面
- 不遗漏重要信息
- 提供足够的细节

### 可理解性

- 结论必须简洁明了
- 使用用户容易理解的语言
- 避免专业术语过多

### 可验证性

- 所有来源必须提供链接
- 所有数据必须有时间戳
- 用户可以自行验证

  ↓

```
