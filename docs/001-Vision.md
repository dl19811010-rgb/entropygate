## 设计原则（Design Principles）

> 所有设计决策必须遵循以下原则。

| 原则 | 说明 |
|------|------|
| **Question First** | 先有问题，再有数据模型，最后建表。问题驱动一切。 |
| **Fact First** | 新闻拆成事实，事实组成事件。事实是原子单位。 |
| **Capability First** | Capability 是核心资产，不是 Event，不是 Article。 |
| **Knowledge First** | 知识积累是平台的核心竞争力，内容只是副产品。 |
| **Evidence Driven** | 所有论断都必须有证据支撑，可追溯到原始来源。 |
| **Official Source Preferred** | 官方来源优先，社区讨论需要交叉验证。 |
| **Human Review** | AI 做结构化和评分，人做最终判断和修正。 |
| **Evolution Oriented** | 记录变化，追踪演化，而不是静态快照。 |

---

## 为什么存在

### 问题

AI 领域的信息环境正在崩溃：

1. **信息碎片化**：模型更新、能力变化散落在几十个渠道——官方博客、GitHub Release、Twitter、Reddit、媒体、API Changelog
2. **信噪比低**：大量融资、合作、采访等商业新闻淹没了真正的能力变化
3. **滞后性**：普通用户往往在能力发布几天甚至几周后才知道
4. **不可追踪**：同一个模型的能力演进没有时间线，无法对比
5. **同质化严重**：所有新闻站都在做"RSS → AI 总结 → 发文章"，内容一模一样

### 机会

没有人在做这件事：

> **系统性地、实时地、结构化地追踪 AI 能力的变化。**

而这件事的价值是指数级的——随着时间推移，积累的能力图谱会越来越完整，平台价值会越来越高。

### 我们的价值公式

```
平台价值 = （能力覆盖度 × 变化发现速度 × 事实准确率） × 知识积累年限
```

## 竞争优势

我们的竞争优势不是代码、不是网站、不是 AI。

而是以下三个不可复制的核心资产：

### 第一：统一的语言规范（AIS）

所有 AI 系统、所有数据、所有输出，都遵循同一套语言规范。

这意味着：
- 不会出现"Memory""Projects""MCP"混在一起无法统计的情况
- 跨模型、跨产品的对比成为可能
- 新加入的 AI 不需要重新学习，直接遵循标准

这是其他竞争对手最难以复制的——语言规范需要长期积累和统一维护。

### 第二：演化数据库（Evolution Database）

我们记录的不是静态的"能力列表"，而是动态的"能力演化"。

例如：
```
Memory
  ↓
  2024-02  OpenAI 首次推出
  ↓
  2024-08  Claude 跟进
  ↓
  2025-01  Gemini 支持
  ↓
  2025-05  Qwen 支持
```

这种演化数据一旦积累，就形成了无法跨越的壁垒——后来者无法回溯历史。

### 第三：问题驱动的知识图谱（AKG）

我们的数据库不是从 ER 图开始设计的，而是从 100 个核心情报问题开始的。

这意味着：
- 数据库的每一张表、每一个字段，都是为了回答具体问题而设计的
- 不会有"为了建表而建表"的冗余数据
- 产品功能直接对应情报问题，不会做无用功

## 核心流程

```
Collect（采集）
  ↓
Normalize（标准化）
  ↓
Extract Fact（提取事实）
  ↓
Validate（验证）
  ↓
Correlate（关联）
  ↓
Build Event（构建事件）
  ↓
Update Capability（更新能力）
  ↓
Update Knowledge Graph（更新知识图谱）
  ↓
Generate Content（生成内容）
  ↓
Generate Answer（生成回答） ← 终点
```

### 为什么 Answer 是终点

传统新闻站的终点是"发布文章"。

而我们的终点是"回答问题"。

文章只是回答问题的一种方式。用户还可以通过：
- 搜索直接获取答案
- 时间线查看演化
- 对比页面进行比较
- API 获取结构化数据

### 核心概念

| 概念 | 定义 | 价值 |
|------|------|------|
| **Signal** | 原始信号，未经加工的信息 | 原材料 |
| **Fact** | 从信号中提取的单个事实 | 原子单位 |
| **Event** | 由多个事实组成的能力变化事件 | 核心记录 |
| **Capability** | AI 系统具备的某种具体能力 | **核心资产** |
| **Knowledge** | 从事件中归纳的高层信息 | 竞争壁垒 |
| **Content** | 面向用户的内容输出 | 表现形式 |
| **Answer** | 对具体问题的结构化回答 | 最终目标 |

## 北极星指标

> **成为 AI 能力变化最快、最准、最完整的发现平台。**

拆分为可量化指标：

| 指标 | 定义 | 目标 |
|------|------|------|
| 官方能力更新覆盖率 | 官方发布的能力变化中，被平台捕获的比例 | ≥95% |
| 官方更新发现延迟 | 从官方发布到平台检测到的时间 | ≤30分钟 |
| 能力类事件占比 | 所有事件中，与能力变化相关的事件占比 | ≥80% |
| 无价值商业新闻占比 | 融资、上市、CEO采访等新闻占比 | ≤5% |
| 重复事件合并准确率 | 同一事件的多条信号被正确合并的比例 | ≥95% |
| 能力标签识别准确率 | 事件中涉及的能力被正确打标签的比例 | ≥90% |

## 总结

我们正在构建的不是一个网站，而是一个**AI 情报操作系统**。

它的核心是：
1. **统一的语言规范**——让所有 AI 理解同一个 AI 世界
2. **演化数据库**——记录能力如何出现、扩散、演化
3. **问题驱动的知识图谱**——从问题反推数据模型，确保每一条数据都有价值

新闻只是其中一个前端。

真正的价值来自于这套能够持续理解和解释 AI 行业演化的知识系统。
