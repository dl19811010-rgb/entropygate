## 深度检查与修复（第三轮）

### 发现并修复的问题
1. **Admin API baseURL 硬编码 HF Space**：`ai-news-admin/src/api/request.js` 生产环境指向 `https://dl1010-entropygate.hf.space/api/v1`，改为同源 `/api/v1`
2. **Admin Vue Router 缺少 base**：`createWebHistory()` → `createWebHistory('/admin/')`，否则路由在 `/admin/` 子路径下全部失效
3. **Admin Vite 缺少 base**：添加 `base: '/admin/'`，确保构建后的静态资源路径正确
4. **SPA 路由不支持**：后端用 `StaticFiles` 挂载无法处理前端路由（如 `/article/123`），改为 catch-all 路由返回 `index.html`
5. **API 404 被 SPA 拦截**：`/{full_path:path}` 捕获了 `/api/v1/nonexistent`，添加 `api/` 前缀检查返回 JSON 404
6. **Admin API 路径全面不匹配**：
   - `/admin/auth/login` → `/admin/login`
   - `/admin/auth/me` → `/admins/me`
   - `/admin/admins/*` → `/admins/*`
   - `/admin/roles/*` → `/roles/*`
   - `/articles/grouped-review` → `/articles/review/grouped`
7. **Token 字段名不匹配**：前端 `res.data.access_token` → `res.data.token`
8. **401 重定向路径**：`/login` → `/admin/login`
9. **Admin.to_dict() 缺少 is_super**：添加 `"is_super": self.role == "super_admin"`
10. **seed_deploy.py 的 status 字段 bug**：`status="active"` → `is_active=1`
11. **Dockerfile.cnb 缺少 admin dist**：添加 `COPY ai-news-admin/dist/`
12. **start_app.sh 使用旧种子脚本**：改用 `seed_complete.py`
13. **restart_service.sh 删除数据库**：移除 `rm -f` 命令，修复 API 路径
